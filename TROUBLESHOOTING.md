# 视频分析故障排除

## 问题：未检测到视频输入

### 可能原因

1. **模型不支持视频** - Kimi K2.5 可能只支持图片，不支持视频
2. **视频编码问题** - 视频格式或编码不被支持
3. **视频太大** - 虽然文件不大，但可能帧数太多
4. **API 参数错误** - videoUrl 格式不正确

### 解决方案

#### 方案 1：使用支持视频的模型

尝试使用 Kimi-Vision 或其他支持视频的模型：

```bash
python analyze.py --video_dir sources_video --model kimi-vision --one_file_per_video
```

#### 方案 2：将视频转换为关键帧

如果模型只支持图片，可以将视频转换为关键帧图片，然后分析图片。

#### 方案 3：检查视频格式

确保视频是 MP4 格式，H.264 编码：

```bash
# 使用 ffmpeg 重新编码
ffmpeg -i input.mp4 -c:v libx264 -preset fast output.mp4
```

### 当前状态

根据错误信息 `未检测到视频输入，无法执行视觉分析`，模型可能：
- ✅ 接收到了请求
- ❌ 但无法解析视频数据

### 建议

1. **先测试小视频** - 创建一个 5-10 秒的短视频测试
2. **检查模型文档** - 确认 Kimi K2.5 是否支持视频
3. **使用图片替代** - 如果模型只支持图片，提取关键帧分析

### 替代方案

如果视频分析不可行，可以考虑：
1. 提取视频关键帧（每秒 1 帧）
2. 将关键帧作为图片输入模型
3. 模型分析多张图片后生成完整描述
